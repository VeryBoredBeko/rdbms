# Инструкция по работе с FastAPI и PostgreSQL

## Цель
Прежде чем приступать к выполнению задания `tech-specification.md`, необходимо ознакомиться с основами FastAPI и научиться создавать первую API с подключением к базе данных PostgreSQL.

---

## Шаг 1. Ознакомление с официальным туториалом

Пройдите раздел официального туториала FastAPI:

[FastAPI Tutorial: SQL Databases](https://fastapi.tiangolo.com/ru/tutorial/sql-databases/#run-the-app-again)

**Что нужно сделать:**
1. Прочитать раздел и понять, как работает FastAPI с базой данных через SQLAlchemy.  
2. Выполнить команды из туториала на своём компьютере:  
   - Создание виртуального окружения (рекомендуется)  
   - Установка зависимостей (`fastapi`, `uvicorn`, `sqlalchemy`, `psycopg2-binary`, `pydantic`)  
   - Настройка подключения к PostgreSQL  
   - Создание моделей таблиц и Pydantic-схем  
   - Создание первых маршрутов GET и POST  
3. Убедиться, что API работает и доступно через Swagger (`http://127.0.0.1:8000/docs`)  

---

## Шаг 2. Практика

Создайте **простую API с одной таблицей**, например `student` с полями:
- `student_id`  
- `first_name`  
- `last_name`  
- `email`  

Реализуйте **эндпоинты**:
- GET `/students` — получить всех студентов  
- POST `/students` — добавить студента  

Проверьте работу API через Swagger или Postman.  

---

## Шаг 3. Переход к основному заданию

После того как вы успешно создали первую API и разобрались с подключением к PostgreSQL:

1. Откройте `tech-specification.md`.  
2. Создавайте модели SQLAlchemy для всех таблиц из ТЗ.  
3. Реализуйте CRUD-эндпоинты для каждой таблицы.  
4. Используйте знания, полученные из туториала, для построения API.  

---

## Совет

Не пытайтесь сразу делать всё ТЗ. Сначала повторите туториал, создайте маленькую рабочую API, а потом масштабируйте проект на все таблицы и эндпоинты.
