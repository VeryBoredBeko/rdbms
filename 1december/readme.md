# Проект по базам данных: Учебный центр

## Тема
Определение рабочих процессов, Концептуальная модель данных и Схема базы данных для информационной системы учебного центра.

---

## Краткая теория

### 1. Определение рабочих процессов
Рабочий процесс — это последовательность действий, которые выполняются для достижения конкретной цели в системе.  
В учебном центре это, например: регистрация студента, запись на курс, проведение занятия, выставление оценок.

**Принципы:**
- Идентификация участников процесса (студенты, преподаватели, администраторы).  
- Описание последовательности действий и участвующих данных.  
- Определение точек хранения и обновления информации в БД.

---

### 2. Концептуальная модель данных
Концептуальная модель данных (ER-модель) описывает, **какие данные существуют и как они связаны**, независимо от способа их хранения.

**Основные элементы:**
- **Сущности** — объекты (студент, преподаватель, курс).  
- **Атрибуты** — характеристики сущностей (имя, дата рождения, стоимость курса).  
- **Связи** — как сущности взаимодействуют друг с другом (1:1, 1:N, M:N).

---

### 3. Схема базы данных
Схема базы данных — это логическая структура БД, отражающая таблицы, поля, типы данных, ключи и ограничения.

**Основные элементы:**
- Таблицы и поля  
- Первичные и внешние ключи  
- Ограничения (NOT NULL, UNIQUE, CHECK)  
- Связи между таблицами:  
  - **1:1** — каждая запись одной таблицы соответствует только одной записи другой  
  - **1:N** — одна запись таблицы A может соответствовать нескольким записям таблицы B  
  - **M:N** — несколько записей таблицы A могут соответствовать нескольким записям таблицы B, реализуется через промежуточную таблицу  

Схема базы данных позволяет затем **создать физическую БД в PostgreSQL**.
